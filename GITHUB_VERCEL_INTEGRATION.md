# GitHub-Vercel Integration Guide

## Overview

AppStudio now includes comprehensive GitHub and Vercel integration with 2-way sync, auto-deployment, PR management, and webhook support. This guide explains how to set up and use these features.

---

## Architecture

### Components

**GitHub Integration:**
- OAuth-based authentication
- Repository creation and management
- File push with automatic commits
- Pull request management
- Issue tracking
- Webhook handling for push events

**Vercel Integration:**
- Project creation and linking
- Automatic deployments on GitHub push
- Environment variable management
- Deployment status tracking
- Production and preview deployments

**2-Way Sync:**
- GitHub-to-Vercel deployment pipeline
- Sync logging and history
- Conflict detection
- Webhook-based triggers

## Setup Instructions

### 1. GitHub OAuth Setup

**Step 1: Create GitHub OAuth App**
1. Go to GitHub Settings → Developer settings → OAuth Apps
2. Click "New OAuth App"
3. Fill in the application details:
   - Application name: "AppStudio"
   - Homepage URL: `https://appstudio-maa8tsdj.manus.space`
   - Authorization callback URL: `https://appstudio-maa8tsdj.manus.space/api/github/callback`
4. Copy the Client ID and Client Secret

**Step 2: Configure Environment Variables**
```bash
GITHUB_OAUTH_CLIENT_ID=your_client_id
GITHUB_OAUTH_CLIENT_SECRET=your_client_secret
GITHUB_WEBHOOK_SECRET=your_webhook_secret
```

**Step 3: Connect GitHub in AppStudio**
1. Navigate to Settings → Integrations
2. Click "Connect GitHub"
3. Authorize the OAuth app
4. Your GitHub account is now connected

### 2. Vercel Integration Setup

**Step 1: Create Vercel API Token**
1. Go to Vercel Dashboard → Settings → Tokens
2. Click "Create Token"
3. Name it "AppStudio"
4. Copy the token

**Step 2: Configure Environment Variables**
```bash
VERCEL_API_TOKEN=your_api_token
VERCEL_WEBHOOK_SECRET=your_webhook_secret
```

**Step 3: Connect Vercel in AppStudio**
1. Navigate to Settings → Integrations
2. Click "Connect Vercel"
3. Paste your API token
4. Your Vercel account is now connected

### 3. Webhook Configuration

**GitHub Webhooks:**
1. In your GitHub repository settings
2. Go to Webhooks → Add webhook
3. Payload URL: `https://appstudio-maa8tsdj.manus.space/api/webhooks/github`
4. Content type: `application/json`
5. Events: Select "Push events", "Pull requests", "Issues"
6. Secret: Use your `GITHUB_WEBHOOK_SECRET`

**Vercel Webhooks:**
1. In Vercel project settings
2. Go to Integrations → Webhooks
3. Add webhook:
   - URL: `https://appstudio-maa8tsdj.manus.space/api/webhooks/vercel`
   - Events: Deployment events
   - Secret: Use your `VERCEL_WEBHOOK_SECRET`

## Features

### Auto Repository Creation

When you generate a new project in AppStudio:

1. **Create Repository**
   ```typescript
   const result = await trpc.github.createRepository.mutate({
     projectId: 30001,
     repoName: "my-app",
     description: "Generated by AppStudio",
     isPrivate: true,
   });
   ```

2. **Push Scaffold**
   ```typescript
   const result = await trpc.github.pushScaffold.mutate({
     projectId: 30001,
     repoName: "my-app",
     commitMessage: "Initial commit from AppStudio",
   });
   ```

3. **Link Vercel Project**
   ```typescript
   const result = await trpc.vercel.createProject.mutate({
     projectId: 30001,
     projectName: "my-app",
     framework: "nextjs",
     gitHubRepoId: 123, // Optional: link GitHub repo
   });
   ```

### Auto Deployment

When you push to GitHub:

1. **Webhook Trigger**
   - GitHub sends push event to `/api/webhooks/github`
   - AppStudio verifies the signature
   - Finds linked Vercel project

2. **Deployment**
   - Vercel client triggers deployment
   - Deployment tracked in database
   - Status updates via Vercel webhook

3. **Status Tracking**
   ```typescript
   const status = await trpc.vercel.getDeploymentStatus.query({
     deploymentId: "dpl_abc123",
   });
   ```

### Pull Request Management

**List PRs:**
```typescript
const prs = await trpc.github.listPullRequests.query({
  projectId: 30001,
  state: "open",
});
```

**Create PR:**
```typescript
const pr = await trpc.github.createPullRequest.mutate({
  projectId: 30001,
  title: "Add new feature",
  description: "This PR adds...",
  sourceBranch: "feature/new-feature",
  targetBranch: "main",
});
```

### Issue Management

**List Issues:**
```typescript
const issues = await trpc.github.listIssues.query({
  projectId: 30001,
  state: "open",
});
```

**Create Issue:**
```typescript
const issue = await trpc.github.createIssue.mutate({
  projectId: 30001,
  title: "Bug: Login not working",
  description: "Users cannot log in with...",
  labels: ["bug", "high-priority"],
  assignees: ["username"],
});
```

### Environment Variables

**Get Variables:**
```typescript
const vars = await trpc.vercel.getEnvironmentVariables.query({
  projectId: 30001,
});
```

**Set Variables:**
```typescript
const result = await trpc.vercel.setEnvironmentVariables.mutate({
  projectId: 30001,
  variables: [
    {
      key: "DATABASE_URL",
      value: "postgresql://...",
      target: ["production", "preview"],
    },
    {
      key: "API_KEY",
      value: "secret_key",
      target: ["production"],
    },
  ],
});
```

## Database Schema

### GitHub Integration Tables

**github_integrations**
- Stores GitHub OAuth tokens and user info
- One per user
- Encrypted access tokens

**project_github_repos**
- Links generated projects to GitHub repositories
- Tracks sync status
- Stores repo metadata

**github_pull_requests**
- Tracks PRs for projects
- Stores PR metadata and status

**github_issues**
- Tracks issues for projects
- Stores issue metadata and labels

### Vercel Integration Tables

**vercel_integrations**
- Stores Vercel API tokens
- One per user

**project_vercel_deployments**
- Links projects to Vercel projects
- Tracks deployment status
- Auto-deploy settings

**vercel_deployment_history**
- Complete deployment history
- Build times and status
- Git metadata

### Sync & Webhook Tables

**github_vercel_sync_logs**
- Tracks all sync operations
- Files changed, added, deleted
- Conflict detection

**webhook_deliveries**
- Webhook delivery history
- Retry tracking
- Payload storage

## Workflow Examples

### Complete Setup Workflow

```typescript
// 1. Connect GitHub
const githubUrl = await trpc.github.getOAuthUrl.query();
// User authorizes and gets redirected

// 2. Connect Vercel
// User provides API token in settings

// 3. Generate Project
const project = await trpc.scaffold.generate.mutate({
  prompt: "Create a Next.js blog...",
});

// 4. Create GitHub Repo
const repo = await trpc.github.createRepository.mutate({
  projectId: project.id,
  repoName: "my-blog",
});

// 5. Push Scaffold
await trpc.github.pushScaffold.mutate({
  projectId: project.id,
  repoName: "my-blog",
});

// 6. Create Vercel Project
const vercelProject = await trpc.vercel.createProject.mutate({
  projectId: project.id,
  projectName: "my-blog",
  gitHubRepoId: repo.id,
});

// 7. Set Environment Variables
await trpc.vercel.setEnvironmentVariables.mutate({
  projectId: project.id,
  variables: [
    { key: "DATABASE_URL", value: "..." },
  ],
});

// 8. Trigger Initial Deployment
await trpc.vercel.triggerDeployment.mutate({
  projectId: project.id,
  production: true,
});
```

### Auto-Deployment Workflow

```
1. Developer pushes to GitHub main branch
   ↓
2. GitHub sends webhook to /api/webhooks/github
   ↓
3. AppStudio verifies signature and finds linked Vercel project
   ↓
4. Vercel client triggers deployment
   ↓
5. Deployment tracked in vercel_deployment_history
   ↓
6. Vercel sends webhook to /api/webhooks/vercel
   ↓
7. AppStudio updates deployment status
   ↓
8. User sees live deployment at production URL
```

## Troubleshooting

### GitHub Connection Issues

**Problem:** "Invalid GitHub signature"
- Solution: Verify `GITHUB_WEBHOOK_SECRET` matches in GitHub settings

**Problem:** "Token expired"
- Solution: Reconnect GitHub OAuth in Settings

### Vercel Connection Issues

**Problem:** "Invalid Vercel API token"
- Solution: Generate new token in Vercel dashboard

**Problem:** "Deployment failed"
- Solution: Check deployment logs in Vercel dashboard

### Webhook Issues

**Problem:** "Webhook not triggering"
- Solution: Check webhook delivery logs in GitHub/Vercel settings

**Problem:** "Signature verification failed"
- Solution: Verify webhook secret matches environment variable

## Security Considerations

1. **Token Storage:** All tokens are encrypted in database
2. **Signature Verification:** All webhooks are cryptographically verified
3. **Rate Limiting:** Implement rate limits on webhook endpoints
4. **Scope Limitation:** GitHub OAuth requests minimal necessary scopes
5. **Audit Logging:** All integrations are logged for audit trail
